# Avignon Activity Entry Guidance

Updated: 2026-05-18

This is the standing Avignon guidance for Portal activity creation and activity follow-up.

- If Sonat asks Avignon to create a Portal activity tied to a distributor, use the distributor contact as the activity contact by default.
- If Sonat names a different account, use the named account instead of the distributor default.
- If the activity is about a distributor contact, Sonat should usually say so explicitly. If the account, contact, or attachment target is unclear, ask Sonat immediately before creating the activity.
- When an activity request is blocked for any reason, Sonat must be told right away in plain English.
- A blocked activity request should say what is missing, what Avignon already confirmed, and the next step needed to finish the Portal entry.
- Do not guess at account or contact matching when Sonat has not specified the target clearly enough to avoid a bad CRM write.
