# Shared Inbox-Thread-First Rule For Direct-Owner Follow-Ups

Status: active shared directive
Owner: AI Workspace Task Manager / Email Coordinator
Applies to: Frank, Avignon, and other approved email-worker personas handling direct-owner follow-up questions on existing threads
Created: 2026-05-21

## Rule

When a direct owner replies on an existing thread with a follow-up question, correction, or "look at this thread" instruction, the worker must check the live inbox/thread context first before asking the owner to restate the question.

If the thread context is present and clear:

- answer directly from the thread when the answer is safely available, or
- route the substantive work and send the completion or blocker report when that work finishes.

If the thread context is missing, incomplete, or blocked:

- say exactly what is missing, such as the prior body, attachment packet, or worker proof, and
- ask only the smallest plain-English question needed to continue.

## What This Prevents

- bouncing an owner question back to the owner when the mailbox already has the needed thread context;
- generic "what is this about?" replies on direct-owner follow-up threads;
- blocker emails that omit the real missing packet or exact missing field.
